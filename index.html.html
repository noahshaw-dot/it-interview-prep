Core concepts to know

Networking Basics

What every IT Support should know

  • IP addressing (IPv4) — e.g. 192.168.1.x is a private local IP
  • DNS = translates domain names to IP addresses (like a phone book)
  • DHCP = automatically assigns IP addresses to devices on a network
  • TCP/IP = the core protocol suite the internet runs on
  • Ping & ipconfig/ifconfig = tools to diagnose network issues
  • LAN (local) vs WAN (wide/internet)
Operating Systems

Windows & basics

  • Task Manager — check CPU, memory, disk usage and kill processes
  • Event Viewer — logs of errors and warnings on the system
  • Device Manager — manage/update drivers for hardware
  • Control Panel / Settings — change system config, users, printers
  • Safe Mode — boots Windows with minimal drivers for troubleshooting
  • CMD commands: ipconfig, ping, tracert, netstat, sfc /scannow
Hardware

Physical components

  • CPU (processor), RAM (temporary memory), HDD/SSD (storage)
  • Motherboard = connects all components
  • GPU = graphics processing
  • POST = Power-On Self Test (what happens when you boot)
  • BIOS/UEFI = firmware that initialises hardware before OS loads
Ticketing & Support

Service desk essentials

  • ITIL basics — Incident, Problem, Change management
  • Ticketing tools: Jira Service Desk, Freshdesk, ServiceNow, Zendesk
  • SLA = Service Level Agreement (time targets for resolving issues)
  • Escalation — know when to escalate to L2/L3 support
  • Remote support tools: AnyDesk, TeamViewer, Remote Desktop (RDP)
Security Fundamentals

IT Security basics

  • Password policies — complexity, expiry, multi-factor auth (MFA)
  • Phishing — fraudulent emails trying to steal credentials
  • Malware types: virus, ransomware, spyware, trojan
  • Principle of least privilege — users get only what they need
  • VPN = encrypted tunnel to connect remotely to a network

Technical questions

Technical · Classic

A user says "my computer is slow." What do you do?

Walk through a structured approach:

  • Ask when it started and if anything changed recently
  • Check Task Manager for high CPU, RAM or disk usage
  • Look for too many startup programs (msconfig or Task Manager > Startup)
  • Check for malware using antivirus scan
  • Check disk space — low storage slows machines down
  • Suggest clearing temp files, or restarting if not done recently
  • If hardware is old, may recommend RAM upgrade or SSD
Technical · Network

A user can't connect to the internet. How do you troubleshoot?

Follow a logical top-down approach:

  • Ask: is it just them or others too? (isolates issue scope)
  • Check physical: is the cable plugged in / WiFi connected?
  • Run ipconfig — do they have a valid IP? (169.x.x.x = no DHCP)
  • Ping 8.8.8.8 — if works, issue is DNS not connectivity
  • Try a different browser or device to narrow down
  • Restart router/switch if appropriate
  • Check if proxy or VPN settings are interfering
Technical · Password

A user is locked out of their account. What do you do?

  • Verify the user's identity first (never skip this step — it's a security risk)
  • Check if account is locked in Active Directory or the system
  • Unlock and/or reset password following company policy
  • Advise them on password policy (length, complexity)
  • Log the ticket appropriately
Technical · Printer

A printer isn't printing. How do you fix it?

  • Check the printer is powered on and online (not paused or offline)
  • Clear the print queue — stuck jobs block everything
  • Restart the Print Spooler service in Windows
  • Check cables / WiFi connection to the printer
  • Verify ink/toner levels and paper
  • Reinstall or update printer drivers if needed

Role & situational questions

Situational

What does IT Support actually do day-to-day?

IT Support Officers typically:

  • Respond to and resolve user tickets (hardware, software, network issues)
  • Set up and configure computers, accounts and peripherals
  • Maintain and monitor IT infrastructure
  • Manage user accounts in Active Directory
  • Ensure security policies are followed
  • Document solutions and update the knowledge base
  • Escalate complex issues to senior engineers
Situational

What is Active Directory and what would you use it for?

Active Directory (AD) is Microsoft's directory service for managing users, computers and permissions in an organisation. As IT Support you would use it to:

  • Create, disable, unlock or reset user accounts
  • Assign users to groups for access control
  • Apply Group Policies (GPO) to enforce security settings
  • Manage computer accounts on the domain

Framework · Must Know

The STAR Method

Use this structure every time you answer a behavioural question.

S
Situation

Set the scene. What was the context? Keep it brief.

T
Task

What was your role or responsibility in that situation?

A
Action

What did YOU specifically do? Use "I", not "we".

R
Result

What was the outcome? Quantify if you can. What did you learn?

Common behavioural questions

Behavioural · Classic

"Tell me about yourself."

Use a 60-second structure: Who you are → What you've done/learned → Why this role.

  • Who: "I'm a self-taught IT enthusiast who's been building my skills through hands-on projects and online learning."
  • What: Mention specific things you've actually done — set up networks, built PCs, fixed devices, completed courses (CompTIA A+, Google IT etc.)
  • Why: "I'm excited about this role because I want to apply those skills in a professional environment and grow in IT support."
Behavioural · Problem Solving

"Describe a time you solved a difficult technical problem."

As a self-taught person, draw on personal projects:

  • Fixing a computer for a family member or friend
  • Troubleshooting your own network or device issues
  • A project where something broke and you had to research a fix

Structure it with STAR — focus on your thought process, what you researched, and the outcome.

Behavioural · Customer Service

"How do you handle a frustrated or impatient user?"

  • Stay calm — never match their frustration
  • Listen fully before speaking — let them feel heard
  • Acknowledge their frustration: "I understand this is affecting your work, let me help fix it."
  • Give them updates so they know progress is happening
  • Focus on the solution, not the blame
Behavioural · Honesty

"What do you do when you don't know the answer?"

This is a trick question — they want honesty, not overconfidence. Say something like:

  • "I'd be honest with the user and tell them I'll find the answer."
  • "I'd check documentation, search for the issue online, or ask a colleague."
  • "I believe in escalating when needed rather than guessing and making things worse."
  • Show that you're a learner — self-taught people are good at googling!
Behavioural · Growth

"Why IT Support, and where do you see yourself in 3 years?"

  • Be genuine — you've been self-teaching, so share what pulled you into IT
  • Show ambition: "I want to build a strong foundation in support, then grow into networking/cloud/security/sysadmin."
  • Mention certifications you're working toward (CompTIA A+, Network+, Google IT Certificate etc.)
  • Tie it back to the company — show you want to grow WITH them
Pro Tip · Questions to Ask Them

Always have 2–3 questions ready for the interviewer

  • "What does a typical first week look like for someone in this role?"
  • "What tools and ticketing systems does the team use?"
  • "What does the team look like and who would I be supporting?"
  • "Are there opportunities for professional development or certifications?"

The Night Before

Prepare tonight

  • Research the company (what they do, size, industry)
  • Re-read the job description — match your skills to their requirements
  • Prepare 2–3 personal examples using STAR method
  • Review the Technical & Q&A tabs on this page
  • Prepare your questions to ask the interviewer
  • Plan your outfit — smart/business casual
  • Confirm interview time, location or video link
  • Get a good night's sleep
The Day Of

Interview day

  • Eat a proper meal beforehand
  • Arrive / log on 10–15 minutes early
  • Bring CV/resume printed (or have it open on phone)
  • Have a notepad and pen for notes
  • Phone on silent
  • Breathe — slow down before answering questions
  • Smile, make eye contact, be confident
Mindset Reminders

Remember this going in

  • Being self-taught is a STRENGTH — it shows initiative and drive
  • You don't need to know everything — show how you THINK and learn
  • They're also being interviewed by you — assess if it's a good fit
  • If you don't know, say "I'd look that up / ask a colleague / research it"
  • Enthusiasm and attitude matter as much as technical knowledge at entry level
