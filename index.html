Core concepts to know

⭐ Healthrite Priority · POS Systems

Point of Sale — what you must know

  • What it is: The checkout system — hardware (terminal, receipt printer, card reader, barcode scanner) + billing/inventory software
  • Common issues: Won't connect to network, card reader failing, receipt printer jammed, software crash, slow transactions
  • Troubleshoot: Check LAN cable/network → restart POS software → check card reader drivers → clear printer queue → escalate if hardware fault
  • Why it matters: POS downtime = zero sales. A 24/7 pharmacy cannot afford this — speed of resolution is everything
  • Tip: Ask them what POS software/brand they use — shows initiative and sets you up to prep more
⭐ Healthrite Priority · Server Administration

Server Admin basics you should know

  • What a server does: Central machine that stores data, runs apps, manages user accounts and network resources for the whole organisation
  • Windows Server: Most common in SMEs — manages Active Directory, file sharing, print services, DNS/DHCP
  • Key tasks: Create/manage user accounts, monitor server health (CPU, RAM, disk), apply updates, manage backups
  • Backups: Critical in pharmacy — patient and inventory records must be backed up daily. Know: local vs cloud vs offsite backup
  • Uptime monitoring: Check Windows Event Viewer logs and respond to alerts before they become outages
⭐ Healthrite Priority · Cybersecurity

Security in a pharmacy environment

  • Pharmacy data is sensitive — prescription records, customer PII, payment data are all targets
  • Password policies — complexity requirements, expiry, MFA (multi-factor authentication)
  • Phishing — fraudulent emails targeting staff credentials, especially common in retail
  • Ransomware can encrypt all data including patient records — backups are the primary defence
  • Principle of least privilege — cashier doesn't need admin rights on the server
  • PCI-DSS awareness — card payment data has strict legal security requirements
  • Physical security — who has access to the server room? Are terminals locked when unattended?
Networking Basics — LAN/WAN

Network fundamentals for Healthrite

  • LAN = Local Area Network (all devices inside the store connected together)
  • WAN = Wide Area Network (connects branches to each other or to the internet)
  • IP addressing — every device needs an IP. Private: 192.168.x.x or 10.x.x.x
  • DNS = translates domain names to IPs. DHCP = automatically assigns IPs to devices
  • Switch = connects multiple devices on LAN. Router = connects LAN to the internet
  • Key commands: ipconfig, ping 8.8.8.8, tracert, netstat
Operating Systems

Windows — the daily driver

  • Task Manager — check CPU, RAM, disk usage; kill stuck processes
  • Event Viewer — logs of system errors and warnings for diagnosing issues
  • Device Manager — manage and update hardware drivers
  • Safe Mode — boots with minimal drivers, great for troubleshooting
  • Services.msc — manage background services (e.g. Print Spooler, DHCP client)
  • CMD: ipconfig, ping, tracert, sfc /scannow, net user, netstat
Hardware

Physical components

  • CPU (processor), RAM (temporary memory), HDD/SSD (storage), Motherboard (connects everything)
  • POST = Power-On Self Test — beep codes at startup indicate hardware faults
  • BIOS/UEFI = firmware that initialises hardware before the OS loads
  • POS-specific hardware: receipt printer, barcode scanner, card reader, customer display
Ticketing & Support

Service desk essentials

  • ITIL basics — Incident (something broke), Problem (root cause investigation), Change (planned update)
  • SLA = Service Level Agreement — the time target you must resolve issues within
  • Ticketing tools: Freshdesk, Jira Service Desk, ServiceNow, Zendesk
  • Escalation — know when to move to L2/L3 rather than keep attempting yourself
  • Remote support tools: AnyDesk, TeamViewer, Windows Remote Desktop (RDP)

⭐ Healthrite-specific scenarios

Scenario · POS Down

"The POS terminal at the pharmacy counter has stopped working. What do you do?"

Stay calm — this is high priority. Move quickly and systematically:

  • Check if it's one terminal or all of them — this tells you if it's local or network/server
  • If one terminal: check power, cables, restart the POS application
  • If all terminals: check the network switch and server — likely a central issue
  • Ping the server from the terminal to check connectivity (cmd: ping [server IP])
  • Restart the POS service on the server if network is fine
  • If hardware fault (card reader, printer), swap with a spare unit if available
  • Keep staff updated every few minutes — communication is critical
  • Escalate if not resolved in 10–15 minutes — this is 24/7 retail, every minute counts
Scenario · Network Outage

"The store's internet went down at 2am. How do you respond?"

  • First: is it internet only, or is the whole internal network down? (Can devices ping each other?)
  • Restart the router — wait 2 minutes to see if it reconnects to the ISP
  • Check if ISP is having an outage (call their support line or check status page)
  • Check if POS can operate in offline mode — some systems queue transactions locally
  • If router is faulty, escalate to ISP support or swap the device if a spare is available
  • Document: time the outage started, impact on operations, actions taken, resolution time
Scenario · Security Incident

"A staff member clicked a suspicious email link. What do you do?"

  • Stay calm — don't alarm them, but act fast
  • Disconnect their machine from the network immediately (unplug cable / disable WiFi)
  • Run a full antivirus/malware scan on the machine
  • Reset their password right away — assume credentials may be compromised
  • Check if any sensitive files were accessed or encrypted (ransomware indicator)
  • Report the incident to management — pharmacy data breaches can have legal implications
  • After resolution: educate the staff member, don't shame them — phishing works on everyone

General technical questions

Technical · Classic

"A user says their computer is slow. What do you do?"

  • Ask when it started and if anything changed recently
  • Check Task Manager for high CPU, RAM or disk usage
  • Check startup programs — too many slow boot and performance (Task Manager → Startup)
  • Run an antivirus scan for malware
  • Check disk space — low storage significantly slows machines
  • Clear temp files, restart if not done recently
  • If hardware is old, recommend RAM upgrade or switch to SSD
Technical · Network

"A user can't connect to the internet. How do you troubleshoot?"

  • Ask: is it just them or others too? (scopes the problem)
  • Check physical connection — cable plugged in / WiFi connected?
  • Run ipconfig — valid IP? (169.x.x.x means no DHCP response — network fault)
  • Ping 8.8.8.8 — if this works, it's a DNS issue not a connectivity issue
  • Try a different browser or device to narrow the cause
  • Restart router or switch if appropriate
  • Check proxy or VPN settings that might be interfering
Technical · Account

"A user is locked out of their account. What do you do?"

  • Verify their identity first — never skip this, it's a security control
  • Check if account is locked in Active Directory or the relevant system
  • Unlock and/or reset password following company policy
  • Advise them on password policy (length, complexity, don't reuse old passwords)
  • Log the incident ticket with the action taken
Technical · Printer

"A printer isn't printing. How do you fix it?"

  • Check the printer is powered on and not showing as offline or paused
  • Clear the print queue — stuck jobs block everything behind them
  • Restart the Print Spooler service in Windows (services.msc)
  • Check cable / WiFi connection between PC and printer
  • Check ink/toner and paper levels
  • Reinstall or update printer drivers if nothing else works

Role & knowledge questions

Situational

"What is Active Directory and how would you use it at Healthrite?"

Active Directory (AD) is Microsoft's system for managing users, computers and permissions across an organisation. At Healthrite you'd use it to:

  • Create accounts for new staff and disable them when staff leave
  • Reset passwords and unlock locked accounts
  • Assign users to groups — e.g. pharmacists get different access to systems than cashiers
  • Apply Group Policies (GPO) to enforce security settings across all machines in the store
Situational

"How would you prioritise multiple IT issues coming in at once?"

Prioritise by business impact — always:

  • Critical: POS down, full network outage, server down — revenue and operations are stopped
  • High: Multiple users affected, system that blocks work from happening
  • Medium: Single user issue that slows but doesn't stop their work
  • Low: Cosmetic issues, non-urgent requests, general queries
  • Acknowledge all tickets — let users know you've seen their issue and give an ETA

Framework · Must Know

The STAR Method

Structure every single behavioural answer this way.

S
Situation

Set the scene briefly. What was the context?

T
Task

What was your specific role or responsibility?

A
Action

What did YOU do? Use "I" not "we".

R
Result

What was the outcome? What did you learn?

Behavioural · Customer Service

"How do you handle a frustrated or impatient user?"

  • Stay calm — never mirror their frustration
  • Listen fully before responding — let them feel heard
  • Acknowledge it: "I understand this is affecting your work, let me get it fixed now."
  • Give updates while you work — silence makes frustration worse
  • Focus on the fix, not the cause or blame
  • At Healthrite: a cashier with a broken POS is under real pressure — be fast and reassuring
Behavioural · Honesty

"What do you do when you don't know the answer?"

They want honesty and resourcefulness — not overconfidence:

  • "I'd check documentation, search for the issue, or consult a more experienced colleague."
  • "I believe in escalating when needed rather than guessing and making things worse."
  • Being self-taught means you're naturally good at finding answers — highlight that

"Why IT Support, and where do you see yourself in 3 years?"
